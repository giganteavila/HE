    02.-Conceptos básicos de la seguridad de la información

    Amenazas a la seguridad de la información¶

    Una amenaza es cualquier suceso que pone en riesgo el funcionamiento correcto de un sistema de información y puede causar daño a los activos de la organización. Según la norma UNE 71504:2008 (MAGERIT), se define como una "causa potencial de un incidente que puede causar daños a un sistema de información o a una organización". Las amenazas pueden ser tanto involuntarias, como un desastre natural, o deliberadas, como un ataque DDoS.

    MAGERIT clasifica las amenazas en los siguientes tipos:

    • De origen natural: Desastres como terremotos, inundaciones, entre otros.
    • Del entorno (industrial): Como cortes de suministro o contaminación.
    • Defectos de las aplicaciones: Vulnerabilidades y fallos en el diseño de los sistemas.
    • Causadas por las personas de forma accidental: Errores humanos que pueden facilitar ataques (ej. ingeniería social).
    • Causadas por las personas de forma deliberada: Ataques intencionados por ciberdelincuentes.

    En la seguridad ofensiva y el hacking ético, es esencial estudiar las amenazas relacionadas con errores humanos, defectos de aplicaciones y ataques deliberados.

    Tipos de intrusos y motivaciones¶

    Se pueden clasificar diferentes tipos de intrusos o ciberdelincuentes según sus motivaciones:

    • Cracker: Se especializan en romper sistemas de seguridad para obtener beneficios ilegales, como en los años 80, cuando se creaban cracks y keygens para eludir licencias de software.
    • Phreaker: Utilizan redes telefónicas para acceder a Internet de manera gratuita.
    • Spammer: Envía correos electrónicos masivos para obtener ganancias económicas o saturar sistemas.
    • Lamer: Intrusos con poco conocimiento técnico, conocidos también como wannabe o script-kiddies, que utilizan herramientas ajenas sin comprenderlas.
    • Insider: Trabajadores que cometen ataques internos, motivados por beneficios económicos o venganza.
    • Exempleado: Antiguo empleado con las mismas motivaciones que un insider.
    • Hacktivist: Atacantes que realizan hackeos con fines políticos o sociales, como el grupo Anonymous.
    • State-Sponsored hacker: Ciberdelincuentes que actúan bajo el respaldo de un gobierno, generalmente para espionaje o sabotaje.

    Códigos dañinos: malware¶

    El malware es un software malicioso diseñado para dañar o controlar un sistema informático. Se propaga de diversas maneras, a menudo a través de aplicaciones móviles maliciosas, correos electrónicos o redes comprometidas. Algunos tipos de malware, como el fileless in-memory malware, son difíciles de detectar, ya que solo existen en la memoria durante la ejecución y no dejan rastros en el disco.

    Clasificación del malware¶

    El malware se clasifica según su funcionamiento y los daños que causa:

    • Exploit: Aprovecha vulnerabilidades para comprometer un sistema.
    • Virus: Se replica en otros programas o archivos una vez ejecutado.
    • Ransomware: Cifra los archivos del sistema y exige un rescate para restaurarlos.
    • Spyware: Recoge información confidencial del usuario y la envía al atacante.
    • Adware: Muestra anuncios constantemente en el sistema.
    • Troyano: Se presenta como software legítimo pero introduce una puerta trasera para el control remoto.
    • Rootkit: Proporciona acceso y control total al sistema sin ser detectado.
    • Keylogger: Registra todas las teclas pulsadas para obtener contraseñas u otra información sensible.
    • Gusano: Se propaga a través de redes, infectando otros dispositivos.
    • Bacteria: Se replica sin control para agotar los recursos del sistema.
    • Bomba lógica: Se activa bajo condiciones específicas previamente definidas.
    • Cryptojacking: Utiliza el hardware infectado para minar criptomonedas.

    Breve historia del malware¶

    Algunos eventos importantes en la historia del malware incluyen:

    • Brain (1986): Primer virus que se propagó fuera de un laboratorio.
    • Stuxnet (2010): Malware diseñado para sabotear sistemas industriales críticos.
    • Wannacry (2017): Ransomware que afectó a 200,000 dispositivos, aprovechando vulnerabilidades de Windows.

    Tipos de ataques¶

    Los ataques cibernéticos se pueden clasificar en varias categorías, como se detalla en la Guía de ciberataques del INCIBE:

    Ataques a contraseñas¶

    El objetivo de estos ataques es obtener credenciales de acceso a sistemas. Se utilizan técnicas como ataques de fuerza bruta o diccionarios de contraseñas recopilados en bases de datos filtradas. Para protegerse, se recomienda el uso de múltiples factores de autenticación y gestores de contraseñas seguros, como GuardedBox, que utiliza cifrado de extremo a extremo.

    Ataques de ingeniería social¶

    La ingeniería social utiliza la manipulación psicológica para engañar a las personas y que realicen acciones perjudiciales. Los ataques más comunes incluyen:

    • Fraude online: Engaños a través de plataformas que parecen legítimas.
    • Phishing, vishing y smishing: Suplantación de identidad para obtener información confidencial.
    • Baiting: Uso de cebos para que el usuario descargue software malicioso.
    • Shoulder surfing: Obtención de información confidencial observando a la víctima directamente.
    • Dumpster diving: Recuperación de datos a través de la basura de una empresa.

    Ataques a las conexiones¶

    Existen varios tipos de ataques que comprometen las conexiones de red:

    • DDoS: Sobrecarga de un servicio con un gran número de solicitudes.
    • Man-in-the-Middle (MitM): Interceptación y modificación de la comunicación entre dos partes.
    • Suplantación (spoofing): Falsificación de datos de red para hacerse pasar por otro usuario o sistema.
    • Escucha (sniffing): Interceptación de datos transmitidos en redes no seguras.
    • Redes trampa (Rogue AP): Creación de puntos de acceso Wi-Fi falsos para interceptar información.

    Ataques a la cadena de suministro¶

    Estos ataques se dirigen a los proveedores de hardware o software de una organización para comprometer los productos que esta utiliza. Un ejemplo destacado es el ataque a SolarWinds, que afectó a importantes agencias gubernamentales al incluir malware en el software Orion.

    Ataques invisibles en el código fuente¶

    Este tipo de ataque aprovecha vulnerabilidades en los compiladores y utiliza caracteres Unicode invisibles en el código fuente, lo que permite alterar el comportamiento de un programa sin ser detectado. Un ejemplo es el ataque Trojan Source, descubierto en 2021.

    Vulnerabilidades¶

    Una vulnerabilidad es un fallo en el diseño de un sistema que pone en peligro su funcionamiento. Pueden ocurrir tanto en software como en hardware. Las vulnerabilidades de software suelen deberse a errores de programación o configuración, mientras que las de hardware afectan a componentes físicos como los procesadores.

    Causas y tipos de vulnerabilidades¶

    Las vulnerabilidades pueden tener varias causas, como errores de diseño o fallos en la implementación de criptografía. Ejemplos notables incluyen:

    • Log4Shell (CVE-2021-44228): Vulnerabilidad en la librería Java log4j, utilizada en muchos sistemas.
    • Meltdown y Spectre: Vulnerabilidades en procesadores que permiten acceder a datos restringidos.

    Documentación de vulnerabilidades¶

    Las vulnerabilidades se documentan con un identificador CVE y se evalúan con el CVSS para determinar su gravedad. Además, existen plataformas como Zerodium que pagan por vulnerabilidades zero-day, es decir, fallos que aún no han sido corregidos.
